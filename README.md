# 🤖 LLM Benchmarking & Evaluation Tool (v2.0)

Инструмент для автоматизированного A/B тестирования и оценки ответов больших языковых моделей (LLM) с использованием архитектуры **LLM-as-a-Judge**.

## 🚀 Основные возможности

* **Автоматизация через n8n:** Параллельный опрос нескольких моделей (через OpenRouter) в рамках одного воркфлоу.
* **Отказоустойчивость (Fault Tolerance):** Реализована система повторных попыток (retries) и обработка ошибок API, предотвращающая рассинхронизацию данных.
* **Умное судейство (LLM-as-a-Judge):** Система оценки на основе штрафных баллов (штрафы за галлюцинации, "воду" и неточности).
* **Интерактивная аналитика:** Python-скрипт на базе **Pandas** для генерации сводных HTML-отчетов с дашбордом (Win Rate, средний балл).

## 🛠 Технологический стек

* **Automation:** n8n (Self-hosted/Cloud).
* **Backend/Analytics:** Python 3.x, Pandas.
* **LLM API:** OpenRouter (Gemini, GPT, Claude).
* **Reporting:** HTML5/CSS3, Excel (openpyxl).

## 📈 Эволюция проекта

### v1.0 — Базовый MVP
* Простая пересылка вопросов через n8n.
* Базовая склейка ответов по позиции.
* Простейший текстовый вывод.

### v2.0 — Enterprise-ready (Текущая версия)
* **Fault Tolerance:** Настроены маршруты ошибок и политики перезапуска запросов.
* **Scoring System:** Внедрена матрица штрафов для объективной оценки качества ответов.
* **Analytics Dashboard:** Визуализация результатов в браузере с расчетом статистических метрик качества.

## 📦 Установка и запуск

1. Клонируйте репозиторий.
2. Установите зависимости: `pip install pandas requests python-dotenv openpyxl`.
3. Настройте файл `.env`, добавив ваш `OPENROUTER_API_KEY`.
4. Импортируйте воркфлоу в n8n и запустите его для генерации `file.csv`.
5. Запустите анализ: `python judge.py`.
